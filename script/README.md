# EasyCon协议测试脚本

本脚本用于测试ESP32-BLE5-NSController-Emulator项目中`uart_easycon.c`实现的EasyCon串口协议。

## 功能特性
- HELLO命令握手测试（验证响应）
- HID数据帧发送测试（不验证响应）
- 定期心跳包测试（每5秒发送HELLO命令并验证响应）
- 完整通信稳定性测试（运行指定时长）
- 详细的测试统计和成功率计算
- 支持Windows和Linux系统

## 安装依赖

### 1. 安装Python
确保系统已安装Python 3.6或更高版本。

### 2. 安装pyserial库
```bash
pip install pyserial
```

### 3. 验证安装
```bash
python --version
pip show pyserial
```

## 使用方法

### 1. 连接硬件
- 将ESP32开发板通过USB连接到电脑
- 确保开发板已烧录正确的固件（包含EasyCon协议实现）

### 2. 查看可用串口
```bash
python easycon_tester.py --list-ports
```

### 3. 运行测试

#### 基本测试（默认60秒）
```bash
# Windows系统
python easycon_tester.py --port COM3

# Linux系统
python easycon_tester.py --port /dev/ttyUSB0
```

#### 自定义测试时长
```bash
python easycon_tester.py --port COM3 --duration 30
```

#### 自定义波特率
```bash
python easycon_tester.py --port COM3 --baudrate 115200
```

#### 详细输出
```bash
python easycon_tester.py --port COM3 -v
```

## 测试流程

### 1. HELLO握手
- 发送HELLO命令帧（`0xA5 0xA5 0x81`）
- 验证单片机返回`0x80`（RPY_HELLO）响应
- 握手失败则终止测试

### 2. 主测试循环
测试持续运行指定时长（默认60秒），包含：

#### 心跳包测试（每5秒）
- 发送HELLO命令作为心跳包
- 验证单片机返回`0x80`响应
- 记录发送次数和成功次数

#### HID数据测试（每秒）
- 发送模拟的HID数据帧
- 包含变化的按钮状态和摇杆位置
- 不验证响应，只记录发送是否成功

### 3. 统计报告
测试完成后显示：
- 总测试时长
- 心跳包发送/成功次数
- HID数据发送/成功次数
- 错误次数
- 操作成功率

## 协议说明

### 命令帧
- **HELLO命令**: `0xA5 0xA5 0x81`
- **HELLO响应**: `0x80`（RPY_HELLO）
- **其他命令**: `0xA5 <cmd_code>`

### HID数据帧
- **原始数据**: 7字节（按钮字节0 + 按钮字节1 + HAT状态 + LX + LY + RX + RY）
- **编码方式**: 7-bit packed编码
- **编码后**: 8字节，最后一个字节bit7=1作为结束标记
- **摇杆缩放**: 原始值0-255缩放到0-4095（左移4位）

### HAT方向状态
| 值 | 方向 | 值 | 方向 |
|----|------|----|------|
| 0x00 | 上 | 0x01 | 右上 |
| 0x02 | 右 | 0x03 | 右下 |
| 0x04 | 下 | 0x05 | 左下 |
| 0x06 | 左 | 0x07 | 左上 |
| 0x08 | 中心 | | |

## 故障排除

### 常见问题

#### 1. 无法连接串口
```
错误: 无法连接串口
```
- 检查串口设备名称是否正确
- 确认设备已连接且未被其他程序占用
- 使用`--list-ports`查看可用串口

#### 2. 无响应或响应不正确
```
✗ 未收到响应
✗ 响应不正确: 预期0x80, 实际0xXX
```
- 确认ESP32开发板已烧录正确的固件
- 检查波特率设置（默认115200）
- 验证硬件连接是否正常

#### 3. 权限问题（Linux）
```
权限被拒绝: /dev/ttyUSB0
```
- 将用户添加到dialout组：`sudo usermod -a -G dialout $USER`
- 或者使用sudo运行脚本：`sudo python easycon_tester.py ...`

### 调试模式
使用详细输出模式查看更多信息：
```bash
python easycon_tester.py --port COM3 -v
```

## 开发说明

### 协议实现
- `easycon_protocol.py`: 完整实现了EasyCon协议，包括7-bit packed编码/解码算法
- 与C代码`uart_easycon.c`保持算法一致性

### 扩展测试
如需扩展测试功能，可以：
1. 修改`easycon_tester.py`中的测试逻辑
2. 在`easycon_protocol.py`中添加新的命令支持
3. 使用`utils.py`中的工具函数辅助调试

### 自动化测试
脚本可以集成到CI/CD流程中，通过检查退出码判断测试结果：
```bash
python easycon_tester.py --port COM3 --duration 10
if [ $? -eq 0 ]; then
    echo "测试通过"
else
    echo "测试失败"
    exit 1
fi
```
